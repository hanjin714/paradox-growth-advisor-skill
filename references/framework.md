# 线索不散 Public Framework

## One-Sentence Definition

线索不散，是帮小团队把线上内容带来的开口、咨询、到店、成交和复购节点，先记录成团队能看见的结构，再判断哪些动作适合交给 AI。

## Business Loop

Use this loop:

```text
内容被看见 → 用户开口 → 线索记录 → 团队跟进 → 结果回流 → 复购触发 → 内容再调整
```

Explain it plainly:

```text
内容不是终点，首单也不是终点。
真正的问题是，用户开口以后有没有被接住，接住以后有没有下一步，做完以后有没有回到记录里。
```

## Five Leak Points

1. Content leak: views exist, but users do not comment, ask, click, call, or visit.
2. Lead leak: users ask questions, but no shared record exists.
3. Follow-up leak: records exist, but no owner, next action, or date exists.
4. Review leak: results exist, but do not return to content and team decisions.
5. Repeat-purchase leak: customer cycles exist, but no reminder or reactivation node exists.

## Basic Lead Table

Use this as the public table:

| Field | Meaning |
|-|-|
| Date | When the lead appeared |
| Source | Video, live, comment, message, referral, store visit |
| Original words | Keep the customer's actual words |
| Need | What they seem to want |
| Stage | Asking, comparing, appointment, paid, lost, repeat |
| Owner | Who follows up |
| Next action | The specific next step |
| Next follow-up date | Not “later”; use a date |
| Lead grade | A/B/C |
| Result | What happened after follow-up |

## A/B/C Rules

A lead:

- Clear need.
- Matched service range.
- Willing to continue.
- Has next action.

B lead:

- Interested but missing key information.
- Needs clarification, content nurturing, or a second question.

C lead:

- Outside range.
- Price-only.
- No next action.
- Weak match.

Say:

```text
A/B/C 不是看不起客户，而是保护小团队注意力。
```

## Tool Choice

Use this public rule:

```text
变化多、需要判断，用 Agent。
步骤稳定、每天重复，用 workflow。
要给团队长期使用，做页面、权限、搜索、状态，用 Codex。
还没跑清楚，先人工跑三次。
```

## Seven-Day Public Action

Day 1: Write the current business loop in one sentence.  
Day 2: Record 10 real openings or inquiries.  
Day 3: Keep customer original words and classify the need.  
Day 4: Add owner, next action, and next follow-up date.  
Day 5: Mark A/B/C and note missing information.  
Day 6: Review which content or source produced real openings.  
Day 7: Decide what stays manual and what can become Agent, workflow, or Codex.

## Three Example Scenes

Local store:

```text
Record source, service item, customer question, next visit or replenishment timing.
```

Service team:

```text
Record platform source, customer original words, current stage, owner, and next material needed.
```

B2B small team:

```text
Record company type, pain point, decision person, next meeting, and proposal version.
```

Keep examples scenario-level and easy for different industries to adapt.
