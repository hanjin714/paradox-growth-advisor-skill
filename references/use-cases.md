# Use Cases

Use these public-safe examples to help outsiders understand what the Skill does.

## What This Skill Is

Plain explanation:

```text
悖论增长谋士 Skill 不是帮你一键成交。
它先帮你把客户从看见你、开口问你、被谁接住、有没有下一步、结果有没有回到记录里这条线画出来。
线画清楚以后，再决定哪些地方用 AI，哪些地方先别自动化。
```

## What It Is For

Use it for:

- A local store that posts short videos but cannot tell which content brings real visits.
- A solo new-media shop owner who has comments and messages but no lead record.
- A small service team that has inquiries scattered across platforms, chat, and employee phones.
- A light B2B team that needs to track inquiry source, owner, next meeting, and result.
- A course or community learner who needs a 7-day or 30-day public action plan.

Do not position it as:

- A universal traffic system.
- A guaranteed conversion tool.
- A private-domain sales machine.
- A full CRM replacement on day one.
- A hidden SOP package.

## Beginner Scenario: Solo New-Media Shop

User says:

```text
我一个人做新媒体网店，每天发内容，也有人问，但我不知道怎么判断哪些有用。
```

Skill should answer:

1. Business loop: content is seen, customer asks, inquiry enters a record, need is classified, next action is set, result feeds back to content.
2. Likely leak: openings are not being recorded and classified.
3. First action: record 10 real customer original words before changing content strategy.
4. First table: source content, original words, need, missing info, next action, result.
5. Tool choice: use Agent to classify original words only after manual records exist.

Public-safe phrasing:

```text
先别急着问 AI 帮你写更多内容。
你先要知道，哪条内容让客户开口，客户原话里到底在问什么。
```

## Local Store Scenario

User says:

```text
我是本地门店，抖音也发，到店不稳定，想用 AI 提升到店。
```

Skill should answer:

1. Do not start with an AI posting machine.
2. Record source content, customer question, service item, visit intention, next visit or replenishment timing.
3. Find whether the leak is content opening, store reception, or post-service repeat trigger.
4. Use 7 days to record real source and service preferences.

Public-safe first table:

| Field | Why |
|-|-|
| 来源内容 | Know which content produced an opening |
| 客户原话 | See real customer concern |
| 服务/产品需求 | Match store offering |
| 到店/预约状态 | Know whether online moved offline |
| 下次可能节点 | Find repeat-purchase or revisit trigger |
| 负责人 | Avoid scattered handoff |
| 结果 | Feed back to content and service |

## Small Service Team Scenario

User says:

```text
我们十几个人做服务，每天有咨询，但销售说跟不住，老板也不知道卡在哪里。
```

Skill should answer:

1. First diagnose where leads scatter: platform message, comments, group chat, employee phone, boss memory.
2. Convert “你跟一下” into five facts: source, original words, stage, owner, next action.
3. Use A/B/C to protect attention.
4. Review every night which leads had next actions and which did not.

Public-safe judgment:

```text
不是团队不努力，是线索没有变成团队共同能看见的对象。
```

## Small B2B Team Scenario

User says:

```text
我们做轻量企业服务，客户来源比较杂，想知道怎么用 AI 管线索。
```

Skill should answer:

1. Separate inquiry sorting from solution commitment.
2. AI can summarize original words and missing information.
3. Human should decide price, risk, commitment, and final proposal.
4. If multiple people need search, status, permissions, and pages, then consider Codex-built tooling.

Public-safe boundary:

```text
AI 可以帮你整理和提醒，但不要让它替你承诺结果。
```

## Good User Inputs

Encourage users to provide:

```text
我是做什么的：
客户从哪里来：
客户在哪里开口：
现在谁接：
现在怎么记录：
最容易散的地方：
我想要：诊断 / 7 天计划 / 30 天计划 / 表格 / 云文档
```

If the user gives only a vague sentence, produce a draft and mark assumptions.

## Bad Inputs And How To Handle Them

If the user says:

```text
帮我做一个 AI 自动成交系统。
```

Respond:

```text
先不做自动成交。
我们先看客户从哪里来、在哪里开口、谁接住、下一步是什么。
这条链路清楚以后，才判断哪一段适合用 AI。
```

If the user says:

```text
帮我设计完整私域转化。
```

Respond:

```text
公开版不展开完整承接和转化细节。
我可以先帮你做公开可用的线索记录、分层、跟进和复盘结构。
```
